## ПРО ПРОЄКТ (ABOUT)

Забудьте все, що ви знали про дитячі ігри. **STONE WARS** — це революційний симулятор тактичних бойових дій на руках, що використовує передові алгоритми генерації випадкових чисел для створення унікального ігрового досвіду.

Це не просто "Камінь, Ножиці, Папір". Це епічна сага про виживання, де один невірний рух або затримка на мілісекунду може коштувати вам життя (віртуального).

## РЕЖИМИ ГРИ

### 1. Класика
Миттєвий вхід у бій. Один раунд. Один переможець. Чистий адреналін без права на помилку. Перевірте свою удачу проти нашого передового AI.

### 2. Турнір
Випробування на витривалість. Встановіть ліміт перемог і пориньте у затяжну серію битв.
* **Динамічний скоринг:** Відстежуйте кожне очко.
* **Психологічний тиск:** Чи зможете ви витримати марафон до 10 перемог?

### 3. Битва
**Ексклюзивний режим**, що змінює правила жанру.
* **Процедурна генерація світу:** Шлях довжиною від 20 до 100 км генерується щоразу по-новому.
* **QTE (Quick Time Events):** Вас атакують із засідки! У вас є лише **4 секунд**, щоб прийняти правильне рішення.
* **Система HP:** У вас лише 3 серця. Помилка у виборі контр-атаки або повільна реакція призведуть до поранення.
* **Візуалізація прогресу:** Спостерігайте за своїм героєм на динамічній мапі прямо в консолі.

## ТЕХНІЧНЕ ЯДРО

Цей проєкт базується на монолітній архітектурі класу `Game`, що забезпечує стабільність та розширюваність. Нижче наведено детальний розбір вихідного коду `stone new.py`:

### Game Engine (`class Game`)
Головний контролер, що керує всіма аспектами симуляції.

####  Persistent Data System (JSON)
* **Функції:** `load_stats()`, `save_stats()`, `save_progress()`
* **Опис:** Реалізовано систему локального збереження прогресу через серіалізацію в `stats.json`. Ваші перемоги, поразки та нічиї зберігаються навічно. Навіть після перезавантаження системи ваша слава не зникне.

#### Input Processing & Validation
* **Функція:** `get_player_choice()`
* **Опис:** Захищена система введення даних з обробкою виключень (`try-except`). Гра не впаде, навіть якщо користувач введе некоректні дані — система м'яко попросить повторити введення.

#### Real-Time Combat Logic
* **Функція:** `mode_battle()`
* **Опис:** Найскладніша частина коду. Використовує модуль `time` для розрахунку часу (`time.time() - start_t`). Якщо гравець не встигає ввести команду за 4.0 секунд — спрацьовує тригер поразки.
* **RNG (Random Number Generation):** Використання `random.sample` для унікального розміщення ворогів на карті.

#### UI/UX Rendering
* **Функція:** `clear_screen()`
* **Опис:** Адаптивна очистка буфера консолі (`cls` для Windows, `clear` для Unix), що забезпечує кінематографічну чистоту екрану між сценами.
